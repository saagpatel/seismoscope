# Seismoscope

[![Platform](https://img.shields.io/badge/platform-iOS%2017%2B-blue)](https://developer.apple.com/ios/)
[![Swift](https://img.shields.io/badge/Swift-6.0-orange)](https://swift.org)
[![License](https://img.shields.io/badge/license-MIT-green)](LICENSE)

An iPhone and iPad app that uses the device accelerometer as a real-time seismograph. Raw sensor data passes through a Butterworth filter chain and an STA/LTA event detector, then detected events are correlated against the USGS earthquake catalog. The waveform scrolls as a Metal-rendered ribbon view.

---

## Features

- **100 Hz accelerometer sampling** with high-pass (gravity removal) and 0.1–10 Hz bandpass filtering
- **STA/LTA event detection** — standard seismological algorithm, configurable threshold, device-calibrated on first launch
- **USGS catalog correlation** — automatically queries the USGS FDSN Event API and matches detections within a 10-minute / 500 km window
- **Metal-rendered ribbon display** — GPU-accelerated scrolling waveform with tappable event annotations
- **Per-event detail view** — onset time, peak acceleration, STA/LTA ratio, and USGS earthquake metadata when matched
- **Region presets + custom coordinates** — select a named region or enter lat/lon for USGS search area
- **Low-power mode** — drops sampling to 50 Hz to extend battery life
- **SwiftData persistence** — event log survives app restarts; unresolved correlations are retried (up to 3 attempts)

| Layer | Technology |

## Prerequisites

- Xcode 16 or later
- iOS 17.0+ deployment target
- A physical iPhone or iPad (the accelerometer is unavailable on Simulator; the app falls back to synthetic waveform data automatically)

## Getting Started

```bash
# Clone the repository
git clone https://github.com/saagpatel/seismoscope.git
cd seismoscope

# Open in Xcode (project file generated by XcodeGen — the .xcodeproj is checked in)
open Seismoscope.xcodeproj
```

1. Select your physical device as the run destination.
2. Set your development team in **Signing & Capabilities**.
3. Build and run (`Cmd+R`).

The app requests accelerometer access on first launch. No API keys are required — the USGS FDSN endpoint is public.

## Project Structure

```
Seismoscope/
├── App/                    # Entry point, AppState, device profile loader
├── DSP/                    # AccelerometerPipeline, EventCoordinator, SyntheticDataSource
├── Metal/                  # RibbonRenderer, shaders, texture/label cache
├── Models/                 # SeismicEvent (SwiftData model), RibbonState, RegionPreset
├── USGS/                   # USGSClient, USGSCorrelator, USGS model types
├── Views/                  # SwiftUI views (ribbon container, status bar, settings, event detail)
└── Resources/              # device_profiles.json, regions.json

SeismoscopeKit/             # Local Swift package — DSP primitives
├── Sources/
│   └── SeismoscopeKit/     # HighPassFilter, BandpassFilter, STALTADetector, CircularBuffer
└── Tests/

SeismoscopeTests/           # Unit tests (USGSCorrelator, pipeline integration)
```
